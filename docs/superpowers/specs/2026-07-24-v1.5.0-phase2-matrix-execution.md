# v1.5.0 Phase 2 — Key×Context Matrix: Execution Method

> Owner-approved execution design for v1.5.0 phase 2 (settled 2026-07-24). This document refines
> *how* the phase-2 key×context consistency sweep is run — a deliberately token-conscious method
> that supersedes the "reuse phase 1's workflow shape" guidance in `notes.md` and the
> `2026-07-24-v1.5.0-polish-audit-design.md` §Phase 2 sequencing note. **What** the matrix is and
> **why** phase 2 exists are unchanged; see that parent spec.

---

## Why a new execution method

Phase 1 was thorough but expensive. The owner named two specific pain points to fix here:

- **Sheer agent count** — the ~962-claim fan-out plus a 6-lens reverse sweep spawned far more
  subagents than the work required.
- **Resume/cache waste** — a long-running `Workflow` died mid-run and cache-missed its entire
  verify layer on resume, because agent prompts embedded upstream agent output (re-serialization
  changed the bytes, defeating the cache).

The budget goal is **"sensible, not brute-force"**: no hard token cap, but the fan-out is scaled to
the work and the megaworkflow is abandoned.

## The key insight

Phase 1 and phase 2 are structurally different. Phase 1's 962 claims were scattered across three
documents — genuinely embarrassingly parallel, and huge. Phase 2's behavior all lives in a **small,
connected set of files** (`keys.go`, `selection.go`, `grab*.go`, `caretForm`, the render/hints
paths). A handful of agents reading those shared files can each build a whole matrix slice; there is
no need for one agent per matrix cell.

**Thoroughness does not come from agent count — it comes from method.** The four safeguards below
port phase 1's *discipline* (exhaustive, inspectable, both-directions) without its scale.

## Execution: direct parallel agents, no Workflow

Discovery uses a small number of **direct `Agent` tool calls run in parallel**, not a `Workflow`
script. This is the deliberate fix for the resume/cache pain point: there is no long-running
workflow to die and no resumable state to cache-miss. If one agent fails, only that one is re-run.
Every agent prompt is **fully self-contained** — given its file targets and the doc text to compare
— so no agent depends on another agent's output.

### The five discovery agents (partitioned by surface)

Behavior is dispatched per-surface in the code, so surface-partitioning gives clean, non-overlapping,
self-contained slices:

1. **Task tree** — NORMAL tree nav, SELECT, GRAB on the tree.
2. **Calendar grids** — month + week/day, DRILL, day-range SELECT, GRAB.
3. **Agenda + the three overview panels** (Calendars / Tasks / Agenda lists).
4. **Forms + modals + the RESIZE sub-mode** (`caretForm` NORMAL/DRILL, dropdowns, dialogs).
5. **Global / cross-cutting keys** — counts (`3j`), `gg`/`G`, `Tab`, `:` commands, and
   **mode-badge / help-bar / hint-line accuracy**. This agent owns the *seams* that
   surface-partitioning would otherwise miss (see safeguard 3).

Each agent fills the same fixed schema (below) for its assigned cells and reports divergences with
`file:line` evidence.

## The four thoroughness safeguards

These are what make a five-agent sweep as complete as phase 1's 962-row inventory.

### 1. Enumerate both axes inline first (the exhaustiveness guarantee)

Before any agent runs, the main session builds the fixed grid:

- **Key axis** — the complete key/chord list, taken from the keybindings table **∪** a grep of
  every `SetInputCapture` / `InputHandler` in the code. The code side catches **undocumented keys**,
  covering the reverse direction (program → spec) within the same pass.
- **Context axis** — the complete interaction-context list (modes × surfaces × forms/modals).

The resulting fixed cell set is handed to the agents. A blank cell is then **visible**, exactly like
an `unverified` claim row in phase 1 — "exhaustive is inspectable." No agent can silently drop a row.

### 2. Fixed shared schema (mechanical synthesis)

Every agent fills the same columns so synthesis is a diff, not a re-interpretation:

| Key/chord | Context | Actual behavior (file:line) | Help bar | `:help` | README | Verdict |
|---|---|---|---|---|---|---|

`Verdict` ∈ `holds` / `code-diverges` / `doc-stale` / `inconsistent-across-contexts`.

### 3. A dedicated cross-cutting agent (the seam hunter)

Agent 5 exists specifically because surface-partitioning is blind to "key X means A on the tree but
B on the agenda, unintentionally." It examines the seams — count behavior, `gg`/`G`, `Esc`
layer-unwinding vs the mode-nesting design, and mode-badge/hint truthfulness — across all surfaces
at once.

### 4. One completeness-critic pass at the end

A single agent (the phase-1 reverse-sweep analog, but one agent, not a 6-lens fan-out) receives the
filled matrix + the enumerated grid and asks: *what key/context/doc-surface is unaccounted for or
looks under-examined?* Its hits are the only follow-up discovery work.

## Verification and triage

- Divergences are verified **inline by the main session** against the cited `file:line`; there
  should be few, and each is cheap to confirm directly. A skeptic agent is spawned **only** for a
  finding that is genuinely ambiguous after an inline look — not by default for every finding (this
  is the redundant-verification cost phase 1 paid that we are not repeating).
- Confirmed divergences go to the **owner in batches**, each with a recommended resolution (fix
  code vs fix doc). The owner signs off before any fix lands.
- Fixes land **repro-first, one commit each, full gate** (code) or grouped doc commits (docs),
  exactly as phase 1 — the cross-cutting rules in the parent spec are unchanged.

## Deliverable

`docs/audit/specdiff/MATRIX.md` — the filled key×context matrix (same ledger discipline as phase
1's `CLAIMS.md`): the enumerated grid, per-cell verdicts with evidence, and a divergence summary
that drives triage.

## The one honest tradeoff

More of the thoroughness now rests on **disciplined synthesis by the main session** rather than on
redundant agents brute-forcing it. The enumerated grid (safeguard 1) is what keeps that honest — a
skipped cell shows blank and cannot hide.

## Estimated cost

~5 discovery agents + 1 completeness-critic + occasional inline-triggered skeptics — a small
fraction of phase 1's fan-out, with no resumable workflow to waste tokens on a mid-run death.
